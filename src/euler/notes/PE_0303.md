# [PE_0303](../PE_0303.java)

$\frac{f(n)}{n}$ is generated from right to left (call it $m$), realizing that for $mn$ to only use digits $\le 2$, then truncating digits from $m$ has to still satisfy that $mn$ only uses digits $\le 2$ in the last $\lfloor log_{10}m \rfloor + 1$ digits. For any iteration, we start with valid $d$-digit $m$-values for which $mn$ only uses digits $\le 2$ in the last $d$ digits. Then we try each multiple with a new leading digit, and save the ones that keep satisfying the requirement for $d+1$. If we keep multiples in an ordered list, we can return the solution to $f(n)$ as soon as $mn$ for a given multiple $m$ only uses digits $\le 2$ in all its digits.
