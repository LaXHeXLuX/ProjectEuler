# [PE_0323](../PE_0323.java)

This is one where I was forced to just accept some simplification steps that are probably valid but which I wasn't able to understand intuitively.

For a $b$-bit string initialized as all zeroes, we need to find the average number of random $b$-bit strings where iterative bitwise OR would yield in the string having all ones.  
For a given bit $b_i$, $0 \le i < b$, the chance it is still $0$ after $n$ iterations is $(\frac{1}{2})^n = 2^{-n}$, and the chance that it has turned to a $1$ is $1 - 2^{-n}$. The chance that all $b$ bits have turned to a $1$ is $(1 - 2^{-n})^b$, and the chance that not all $b$ bits have turned, is $1 - (1 - 2^{-n})^b$.

For some reason, 
$$E[T] = \sum_{n=0}^\infty P(T > n)$$
I cannot explain it intuitively. Once you accept that, it follows that
$$\sum_{n=0}^\infty P(T > n) = \sum_{n=0}^\infty (1 - (1 - 2^{-n})^b)$$
and using the [binomial theorem](https://en.wikipedia.org/wiki/Binomial_theorem#:~:text=A%20simple%20variant%20of%20the%20binomial%20formula%20is%20obtained%20by%20substituting%201%20for%20y), 
$$(1 - 2^{-n})^b = \sum_{k=0}^b \binom{b}{k} (-1)^k 2^{-nk} \Rightarrow$$
$$1 - (1 - 2^{-n})^b = \sum_{k=1}^b \binom{b}{k} (-1)^{k+1} 2^{-nk}$$
Then
$$E[T] = \sum_{n=0}^\infty \sum_{k=1}^b \binom{b}{k} (-1)^{k+1} 2^{-nk}$$
For some reason, you can safely swap the sums:
$$\sum_{n=0}^\infty \sum_{k=1}^b \binom{b}{k} (-1)^{k+1} 2^{-nk} = \sum_{k=1}^b \sum_{n=0}^\infty \binom{b}{k} (-1)^{k+1} 2^{-nk} = \sum_{k=1}^b (-1)^{k+1} \binom{b}{k} \sum_{n=0}^\infty 2^{-nk}$$
I cannot explain that intuitively, either.  
$\sum_{n=0}^\infty 2^{-nk}$ is an [infinite geometric series](https://en.wikipedia.org/wiki/Geometric_series#Infinite_geometric_series_and_convergence). Using the formula 
$$\sum_{n=0}^\infty ar^n = \frac{a}{1-r}$$
with $a = 1$ and $r = 2^{-k}$, we get
$$\sum_{n=0}^\infty 2^{-nk} = \frac{1}{1 - 2^{-k}} = \frac{2^k}{2^k - 1}$$
giving us
$$E[T] = \sum_{k=1}^b (-1)^{k+1} \binom{b}{k} \frac{2^k}{2^k - 1}$$
