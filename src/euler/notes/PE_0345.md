# [PE_0345](../PE_0345.java)
 
To brute-force the problem by iterating all possible matrix sums, we would need to go through $15! = 1307674368000$ permutations.

A more efficient way is to realise that for any column, we only care about the set of used columns so far, not what their specific order was. By going through all possible subsets of columns, we only have $2^{15} = 32768$ permutations, representable with bitmasks.  
By keeping an array of the best matrix sum for each bitmask and iterating through each bitmask, we take advantage of the fact that:
1) The next row to check is equal to the current bit count in the mask.
2) For any bitmask, any sub-bitmasks (e.g. with one less set bit) are already iterated.
In the end, the answer is at index $2^{15} - 1$ of our best sum array.
