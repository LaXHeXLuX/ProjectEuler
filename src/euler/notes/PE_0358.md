# [PE_0358](../PE_0358.java)



Let's start with three assertions:
- a number $n$ is cyclic iff it is the repeating decimal period of $\frac{1}{x}$ for some integer $x$, where $\text{len}(n) = x - 1$
- $\text{p-len}(n) \le \phi(n)$ and $\text{p-len}(n) \le n-1$, where $\text{p-len}(n)$ is the length of $\frac{1}{n}$'s decimal period and $\phi(n)$ is Euler's totient function
- if $\text{p-len}(n) = n-1$, then $\frac{1}{n} = \frac{p}{10^{n-1} - 1}$, where $p$ is the decimal period of $n$

## Assertion 1
I will make no attempt at justifying the first assertion, other than referencing [Wikipedia](https://en.wikipedia.org/wiki/Cyclic_number#Relation_to_repeating_decimals).

## Assertion 2
An integer $n$'s reciprocal decimal starts repeating when the running remainder becomes $1$ (since it is $1$ at the start). Then $\text{period}(n) = \text{ord}_n(10) = k$, where $10^k \equiv 1 \pmod{n}$. Assuming $\gcd(n, 10) = 1$, $n$ will be coprime to every following remainder, and there are only $\phi(n)$ possible remainders below $n$.  
If $\gcd(n, 10) > 1$, write $n = 2^a \cdot 5^b \cdot n_0$. $\text{p-len}(n) = \text{p-len}(n_0)$, because $\text{p-len}(n) = \text{p-len}(2n) = \text{p-len}(5n)$.

## Assertion 3
The third assertion is true for $\text{p-len}(n) = n-1$, because then $\gcd(n, 10) = 1$ and the decimal of $\frac{1}{n}$ will only have the period without leading digits. Then $\frac{p}{10^{n-1} - 1}$ will exactly "fit" $p$ to repeat in the resulting decimal - which is the same as $\frac{1}{n}$.

We can rewrite the equation as 
$$p \cdot n = 10^{n-1} - 1$$
Since we require $p \equiv 56789 \pmod{10^5}$, we can write
$$56789n \equiv -1 \pmod{10^5}$$
Then $n \equiv \text{modDivide}(56789, -1, 10^5) \pmod{10^5}$, and we can iterate only over such values.

## Algorithm
Then we are looking for some $n$ with period $p = 00000000137 \ldots 56789$. This gives us
$$0.00000000137 < \frac{1}{n} < 0.00000000138 \Rightarrow $$
$$0.00000000137n < 1 < 0.00000000138n \Rightarrow $$
$$137n < 10^{11} < 138n$$
which gives limits $\frac{10^{11}}{138} < n < \frac{10^{11}}{137}$.

Iterating through such $n$, we are already guaranteed that $p = 00000000137 \ldots 56789$ if $\text{p-len}(n) = n-1$. Then we compute the period length (which requires factoring $n$), and only if $\text{p-len}(n) = n-1$, we compute the digit sum. 

Per [Midy's theorem](https://en.wikipedia.org/wiki/Midy%27s_theorem), there is an $O(1)$ way to get the digit sum - $9 \cdot \frac{n-1}{2}$. This works because for any period of $\frac{1}{n}$ where $n$ is prime and the period length is even, every digit in the second half is the $9$'s complement of a digit in the first half.
