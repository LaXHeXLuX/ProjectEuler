# [PE_056](../PE_056.java)

Given $L_a, L_b$ we want to find $a^b$ with the largest digit sum such that $a < L_a, b < L_b$.

The solution skips many $(a, b)$ pairs if $a^b$ cannot realistically have a digit sum above the current maximum. Having a current maximum digit count $M$, the minimal digit count for $a^b$ must be higher than $\frac{M}{9}$, so 
$$\log_{10}a^b > \frac{M}{9} \Rightarrow b > \frac{M}{9 \cdot \log_{10}a}$$
However, given a large number of digits in $a^b$, it is also reasonably safe to assume that the average digit value that made up $M$ is only slightly higher than the statistical average $4.5$.  
The precise bound to go for is anyone's subjective guess. I have gone for a somewhat arbitrary dynamic limit $d_\text{max} = 4.5 + \frac{1}{D^{\frac{1}{4}}}$, where $D$ is $L_b \cdot \log_{10}L_a$ (i.e. maximum amount of digits in $a^b$), giving the bound
$$b > \frac{M}{d_\text{max} \cdot \log_{10}a}$$
