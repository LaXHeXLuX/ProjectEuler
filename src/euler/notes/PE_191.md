# [PE_191](../PE_191.java)

Let $D$ be the prize string length.  
Consider prize strings without L's, such as `OOOO`, `OOOA`, `AOAA`. In each of those, you can make another valid prize string by replacing any one O with L, since one L is allowed and this doesn't affect consecutive A's. Also, since we're only replacing O's, any A/O/L string corresponds to exactly one A/O string.   
Then, we can get the final prize string count by generating every binary A/O string, and for each, add the count of O's to the count (+1 for the A/O string itself).  
Additionally, we can assert that the current string ends with O, and then add either O, AO, or AAO, without having to track the current consecutive A's. This requires hardcoded return values for $D < 3$.
