# [PE_301](../PE_301.java)

Per [Wikipedia](https://en.wikipedia.org/wiki/Nim#Mathematical_theory), in a three-heap Nim game, the function $X(n_1, n_2, n_3)$ is bitwise xor applied to the binary representation of the heaps.
In binary, the heaps $n, 2n, 3n$ will follow the pattern:
$$n = \overline{b_1 b_2 \ldots b_l}_2$$
$$2n = \overline{b_1 b_2 \ldots b_l0}_2$$
$$3n = \overline{b_1 b_2 \ldots b_l0}_2 + \overline{b_1 b_2 \ldots b_l}_2$$

If, in the binary representation of $n$, there are never two $1$'s in a row, then the binary addition $n + 2n = 3n$ won't carry over any digits, meaning $3n$ will have exactly the same $1$'s as $n$ and $2n$ combined, and $X(n, 2n, 3n) = 0$.  
Else, when the binary representation of $n$ has two $1$'s in a row somewhere, then we can write it as
$$n = \overline{b_1  \ldots b_{i} 0 1 1 b_{i+4} \ldots b_l}_2$$
(since the binary number isn't infinite, you will always have a $0$ at some point).  
Then, let's go over cases for $b_i$:
- $b_i = 0$. Then $n = \overline{b_1  \ldots 0 0 1 1 b_{i+4} \ldots b_l}_2$ and $2n = \overline{b_1  \ldots 0 0 1 1 b_{i+4} \ldots b_l0}_2$. Adding them up, we see that $3n$'s $b_{i+1}$ must be $1$. The corresponding bits in $n$ and $2n$ are $0$. We have an odd number of $1$'s in that position, meaning $X(n, 2n, 3n) \not= 0$.  
- $b_i = 1$. Then $n = \overline{b_1  \ldots 1 0 1 1 b_{i+4} \ldots b_l}_2$ and $2n = \overline{b_1  \ldots 1 0 1 1 b_{i+4} \ldots b_l0}_2$. Adding them up, we see that $3n$'s $b_{i+1}$ must be $0$. The corresponding bits in $n$ and $2n$ are $1$ and $0$. We have an odd number of $1$'s in that position, meaning $X(n, 2n, 3n) \not= 0$.  

This proves that $X(n, 2n, 3n) = 0$ iff there are never two $1$'s in a row in the binary representation of $n$.

To find all $n$, $n \le 2^L$, where $X(n, 2n, 3n) = 0$, note the recursive formula:
- There is one positive 2-digit binary number which has no repeating ones ($10$).
- From this number, we can create one valid 3-digit binary number which ends in $1$ ($101$), and one which ends in $0$ ($100$).
- From those, we can create one valid 4-digit binary number ending in $1$ for each 3-digit number ending in $0$, and we can create one valid 4-digit binary number which ends in $0$ for each 3-digit number (both ending in $0$ and $1$).
- Letting $f(n)$ signal the total $n$-bit numbers with no repeating ones, and $f_0(n)$ signal the amount of such numbers ending in $0$, we can write $f_0(n) = f(n-1)$ and $f(n) = f_0(n) + f_0(n-1)$, giving us $f(n) = f(n-1) + f(n-2)$ - the Fibonacci sequence.
